# Finishing the Development

When you've finished development of the GUI, you can compile it to a static
resource that can be served directly from Odin Control.

Note there are some potential differences between the static version of the
project, and the version run in development mode:

### Strict Mode

You may have noticed that the `main.tsx/jsx` file of your project wraps everything
in a component called `<StrictMode>`. [Strict Mode](https://react.dev/reference/react/StrictMode)
is a development tool provided by React which helps find common bugs.

Full details are available on the React reference page linked, but the basic
intention of Strict Mode that it runs several parts of the GUI twice to check
for **Purity**. This means that if any Components have any side effects
when rendering, Strict Mode will make it obvious something is wrong.

This is a development-only behaviour and will do nothing in a compiled GUI.

### Environment Variables

Vite projects can import [Environment Variables](https://vite.dev/guide/env-and-mode)
from various files within the project:
- .env
  - loaded in all cases
- .env.local
  - loaded in all cases, ignored by git
- .env.[mode]
  - only loaded in specified mode
- .env.[mode].local
  - only loaded in specified mode, ignored by git

During development, its recommended that the address of the Odin Control Instance
being used is stored in one of these files for Development Mode, ideally a
local only one to avoid potentially sharing IP addresses on Github:

```Dotenv
# .env.development.local
VITE_ENDPOINT_URL=http://localhost:8888

```

This means when building the static GUI, the any AdapterEndpoints will not use
this value
a blank string, allowing the GUI to make requests of the server that is serving
it directly.

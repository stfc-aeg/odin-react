# Getting Started

Odin React was developed with the use of NodeJS and Vite in mind.

<blockquote class="github-alert">It's useful to already have your Odin Control instance running the
adapter(s) you wish you interact with using the GUI, before starting
development of the GUI. Whilst the GUI can run in development mode without
it, any components that try to interact with it will throw errors.</blockquote>

## Project Setup

A [Template](https://github.com/stfc-aeg/odin-react-template) is provided to
set up an Odin React project ready for development quickly and easily.

Navigate to the location which will contain your Odin React Project in a terminal.
You can then use a python library called [Cookiecutter](https://www.cookiecutter.io/)
to copy and initialise the project template, with the following commands:

```bash
# Install the Cookiecutter library into your python environment
pip install cookiecutter

# Using Cookiecutter, create the start of your project using the template
cookiecutter git+ssh://git@github.com/stfc-aeg/odin-react-template
```

This will then prompt you on the command line to select which template to
use, and setup other values for the project. It's recommended you use the
**Typescript** template, which provides stricter variable typing during
development to reduce the chances of unexpected bugs.

The prompts shown by Cookiecutter are:
- **Select a Template**: Select which template to download and setup.
1. **Set the Name of the Project**: The name of the GUI project.
Sets the name in the GUIs Nav Bar.
1. **Set the name of the package**: Will default to a lower case version
of the project name. Sets the name in the `package.json`
1. **Adapter Name**: The name of an adapter to connect to. The template provides
an [Adapter Endpoint](../?path=/docs/components-adapterendpoint--docs) in the 
`App.tsx/jsx` file setup to connect to this adapter. It can be changed after
installation, and additional endpoints can be created if needed.
1. **Project Folder**: the name of the directory to contain the project.
1. **Default Odin Control Endpoint**: the address, along with port number,
that the Odin Control API is accessible on.
1. **Include Plotly**: The plotly package allows the creation of graphs
to display data. It is not a small package and so has been made optional
if it's not required.

Once the project is created from the template, you need to install the dependencies
and run the project in Development Mode.

```bash
# Move into the directory created by the tempalte
cd app  # You may have chosen a different name for the project folder.

npm install  # this step can take a few minutes

npm run dev  # runs the development server, making it available to see in the web browser
```

Once running, the terminal will display the address of the served webpage. Vite
will also montior the codebase and reload the server when changes are made.

```bash
# Terminal Output once running
  VITE v7.0.4  ready in 442 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

Changes made to the codebase should show in that page, so it can
be used to test changes made quickly and easily.
